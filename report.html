01 · Overview

Project Overview

Security tools like antivirus engines, EDR platforms, IPS systems, and firewalls primarily rely on signature-based detection — comparing file or network content against a database of known malicious patterns. This makes unmodified payloads straightforward to catch, but also means that any transformation that breaks the recognizable pattern can defeat detection entirely.

This framework provides a controlled, ethical environment to study exactly those transformations. It implements encoding and obfuscation techniques drawn from real-world red team practice, then simulates signature-based detection to measure which techniques bypass it — and why.

Red Team Value

Understand how attackers mutate payloads to evade AV/EDR scanning in offensive engagements.

MODULE 1 — Encoding

Implements three classical encoding techniques plus multi-layer stacking. Each encoding transforms the payload into a form that contains none of its original keyword sequences.

MODULE 2 — String Obfuscation

Structural transformations that alter payload appearance without classical encoding. These break string matching even when the underlying content is still text-readable.

A 22-entry regex signature database covering common payload keywords. Each variant is checked against all signatures and receives a verdict of BYPASSED or DETECTED.

Key Observations

  • Case swap is detected — because the simulated detector uses case-insensitive matching (re.IGNORECASE), mirroring real-world AV behavior. This is an intentional design choice to keep the simulation realistic.
  • Simple encoding fully bypasses detection — Base64, XOR, and ROT13 all produce output with zero recognizable keywords, defeating the signature set entirely.
  • Multi-layer encoding is most robust — the 3-layer XOR → Base64 → ROT13 chain produces output that is maximally opaque and has no structural similarity to the original.
  • Limitation: static detection only — real-world EDRs also use heuristic, behavioral, and ML-based detection. These techniques would not necessarily evade those layers.
  • String reversal is a weak obfuscation — effective against left-anchored regex but would be caught by a bidirectional or normalized scanner.
